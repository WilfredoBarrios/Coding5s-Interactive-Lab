## 🚀 Як почати?

Для інтерактивної взаємодії з цією лабораторією на вашому комп'ютері має бути встановлений **Livebook**.

1. **Встановіть Livebook:** Завантажте його на [livebook.dev](https://livebook.dev).
2. **Відкрийте лабораторію:** Ви можете переглянути папки нижче та натиснути кнопку **"Run in Livebook"** всередині кожного файлу або безпосередньо імпортувати URL у сесію Livebook.

## 📚 Методологія Coding5s (5 етапів)

Наш підхід базується на **«Керованому когнітивному опорі»**. Кожна лабораторія проведе вас через наступні етапи:

| Етап | Назва | Мета |
|-------|--------|----------|
| **1** | Practice | Написання коду з нуля |
| **2** | Debug | Пошук навмисних логічних помилок |
| **3** | Complete | Заповнення архітектурних прогалин |
| **4** | Refactor | Оптимізація існуючого коду |
| **5** | Extend | Додавання нових функціональних можливостей |

**Цей репозиторій містить Етап 1 (безкоштовний).** Етапи 2–5 доступні у преміум-версії.

## 🧠 Сократівський ментор (AI)

Усередині кожного файлу ви знайдете **«Підказку ментора» (Mentor Prompt)**. Це не звичайний чат-бот, а симулятор елітного наставника, який:

- **Не дасть вам прямої відповіді** (спочатку)
- Ставитиме запитання, щоб ви самі виявили помилку
- Використовує аналогії з реального світу для пояснення концепцій BEAM
- Допомагає позбутися «кодингу навмання» та вибудувати справжню інженерну інтуїцію
- Ставте йому запитання, і він відповість вам на рівні провідного інженера (Senior).

## 🏛️ Архітектура та ментори

Кожна лабораторія включає **Explanation Gate** (інформаційний шлюз), де наші ІІ-ментори (наприклад, *Staff Engineer Auditor*) перевіряють вашу логіку перед переходом на наступний етап.

- **Safe Guide:** Доброзичливий ментор для 1-го етапу
- **Border Guard:** Саркастичний ментор для аудиту коду
- **Незабаром:** Jinpachi Ego (Stateful5s)
